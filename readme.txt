🔹 Moja rekomendacja: Flask + SQLite/MySQL
Myślę, że na początek Flask + SQL to najlepsza opcja, bo:
✅ Możesz łatwo dodać nowe funkcje (np. historię warzeń, statystyki).
✅ Możesz zrobić prosty interfejs w HTML + Bootstrap, a potem ulepszać.
✅ Masz możliwość rozszerzenia na aplikację mobilną lub desktopową w przyszłości.
________________________________________
🔹 Jak to podzielić na etapy?
1️⃣ Podstawowa wersja MVP (Minimal Viable Product):
✅ Model danych (SQLite/MySQL):
•	recipes (przepisy: nazwa, składniki, instrukcja)
•	inventory (magazyn: nazwa składnika, ilość, jednostka)
✅ Funkcje:
•	Dodawanie nowych przepisów.
•	Dodawanie i edytowanie składników w magazynie.
•	Sprawdzanie, co mogę uwarzyć na podstawie dostępnych składników.
✅ Interfejs:
•	Prosty frontend we Flask + HTML (Bootstrap).
